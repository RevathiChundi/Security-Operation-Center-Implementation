# 🛡️ Security Operations Center (SOC) Implementation and Threat Analysis

## 📌 Overview
This project focuses on the **design, implementation, and operation of a Security Operations Center (SOC)** in a simulated enterprise environment. Collaborating as a team, we built an end-to-end security monitoring and incident response pipeline to detect, analyze, and mitigate cyber threats in real time.

The project emphasizes **practical SOC workflows**, including log aggregation, threat detection, incident investigation, threat intelligence enrichment, and vulnerability management using industry-standard tools.

## 🎯 Objectives
- Build and configure a functional SOC environment  
- Monitor and analyze logs from multiple systems in real time

---

## 🏗️ SOC Architecture
The SOC environment was designed with multiple integrated layers:

- **Log Collection Layer:** Centralized ingestion of logs from endpoints and systems  
- **SIEM Layer:** Aggregation, correlation, and visualization of security events  
- **Detection Layer:** IDS/IPS for identifying malicious activities  
- **Incident Response Layer:** Case management and investigation workflows  
- **Threat Intelligence Layer:** Enrichment of indicators using external sources  
- **Vulnerability Management Layer:** Continuous scanning and risk prioritization

### 🔍 SIEM & Monitoring
- **Wazuh** – Log collection, host-based intrusion detection

## 🔄 SOC Workflow

1. **Log Collection** → Wazuh collects logs from endpoints and systems  
2. **Log Aggregation** → Splunk ingests and correlates logs  
3. **Alert Generation** → Detection rules trigger alerts for anomalies  
4. **Incident Creation** → Alerts are pushed to TheHive as cases  
5. **Threat Enrichment** → Cortex analyzes observables using threat intelligence  
6. **Investigation** → Analysts perform detailed analysis and root cause identification  
7. **Response & Mitigation** → Containment, remediation, and reporting
